# 项目协作约束

- 所有用户界面、说明和业务输出使用简体中文；代码、变量和文件名使用清晰英文。
- TypeScript 开启严格模式，不使用无必要的 `any`。
- 修改后必须运行 `npm run lint`、`npm run test`、`npm run build`。
- 不提交密钥、真实客户数据或个人信息；服务端不得打印完整业务输入和模型完整响应。
- 不能为了展示 Agent 而把固定流程强行设计成 Agent。能用规则解决，不使用大模型；能用工作流解决，不使用 Agent；有稳定 API 时不优先使用 RPA。
- 高风险、高金额、调账、删除、合同审批、合规结论必须保留人工控制。
- 页面必须支持空、加载、错误和 Mock 状态，不留影响运行的 TODO、假按钮或未实现菜单。
